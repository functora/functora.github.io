functora-css

A classless, dependency-free, configuration-free, JS-free, mobile-first, responsive, lightweight, semantic pure CSS framework with light and dark themes and a collapsible navbar.

This page is built entirely with semantic HTML elements and styled exclusively by functora-css - no extra styles, no JavaScript.

Appearance

functora-css supports light and dark themes via the data-theme attribute on the <html> element. There are two complementary mechanisms for applying it:

Attribute

Set data-theme="light" or data-theme="dark" directly on <html> to lock the page to a single theme. This is useful when no runtime theme switching is needed - for example in static pages that always use one theme, or in applications where the attribute value is stored and managed externally (localStorage, database, server-side rendering, etc.).

Checkbox

functora-css provides two built-in checkbox IDs that toggle the theme with pure CSS - no JavaScript required. The user chooses which checkbox to use depending on which theme should be the default (unchecked state).

id="data-theme-dark" - default is light. When unchecked the page renders in light theme; when checked it switches to dark.

id="data-theme-light" - default is dark. When unchecked the page renders in dark theme; when checked it switches to light.

Place the chosen checkbox inside the <nav> element (hidden via CSS) and use a <label> element with the corresponding for attribute as the visible toggle button. The navbar contains two hidden checkboxes - one for the mobile menu toggle and one for the theme toggle. Their order matters for CSS sibling selectors: 

<nav>
  <label>
    <input type="checkbox" />
    <input type="checkbox" id="data-theme-dark" />
    <header>...</header>
    <ul>
      <li><label for="data-theme-dark">Theme</label></li>
    </ul>
  </label>
</nav>

The Theme button in the navigation bar above works this way - clicking it toggles the hidden data-theme-dark checkbox and switches between light and dark without any JavaScript.

Navigation

The responsive navbar uses a checkbox-based toggle for mobile menus without any JavaScript. The structure requires a specific order of elements inside a <nav> tag for the CSS selectors to work correctly.

Structure

The navbar must follow this exact structure: a <label> wrapping two hidden checkboxes (mobile menu toggle and theme toggle), a <header> for the brand/logo, and a <ul> for navigation links. Dropdown menus use a nested <menu> element inside a list item. 

<nav>
  <label>
    <input type="checkbox" />
    <input type="checkbox" id="data-theme-dark" />
    <header><a href="#">Site name</a></header>
    <ul>
      <li><a href="#one">Link</a></li>
      <li>
        <a>Dropdown</a>
        <menu>
          <menuitem><a href="#two">Item one</a></menuitem>
          <menuitem><a href="#three">Item two</a></menuitem>
        </menu>
      </li>
    </ul>
  </label>
</nav>

Mobile behavior

On mobile viewports the navigation links are hidden by default. Tapping the header area (or the hamburger icon) toggles the first checkbox, which reveals the menu via pure CSS. The theme toggle remains independent and does not affect the menu state.

Buttons

Buttons are styled using native HTML elements. No classes are needed - use the primary attribute for primary buttons, round for rounded corners, and disabled for disabled state.

Standard buttons

Anchor as button

Use the btn attribute on an anchor element to style it as a button. Combine with primary for primary styling.

Link button Primary link

<a btn primary href="#">Primary link</a>

Forms

Form elements are styled with a clean, minimal look. Labels should be placed directly before their corresponding inputs. Text inputs and textareas default to 100% width.

Text inputs

Text inputs have a subtle border that highlights on focus. Place the <label> directly before the input. 

<label for="name">Name</label>
<input id="name" type="text" placeholder="Your name" />

Textarea

Textareas auto-resize vertically and have the same styling as text inputs. 

<label for="bio">Bio</label>
<textarea id="bio" placeholder="Tell us about yourself"></textarea>

Select

Select dropdowns are styled to match other form elements.

<label for="lang">Language</label>
<select id="lang">
  <option>Rust</option>
  <option>TypeScript</option>
  <option>Python</option>
</select>

Radio and checkbox

Radio buttons and checkboxes are styled with custom indicators. Place the <label> directly after the input. 

<input type="radio" name="radio" id="r1" /><label for="r1">Option A</label>
<input type="radio" name="radio" id="r2" checked /><label for="r2">Option B</label>
<input type="checkbox" id="c1" /><label for="c1">Check me</label>
<input type="checkbox" id="c2" checked /><label for="c2">Already checked</label>

Tooltips

functora-css provides pure CSS tooltips with no JavaScript. Use the tt attribute on most elements to show a tooltip on hover. Tooltips use ::before and ::after pseudo elements, so they work on any element that supports them.

Basic tooltip

Hover me (tooltip) 

<span tt="Hello from functora-css">Hover me</span>

Tooltip on buttons

<button tt="This is a normal button">Hover button</button>
<button primary tt="Primary button tooltip">Primary</button>

Disabled button tooltip

Use the dtt attribute to show a tooltip when a button is disabled. This is useful for explaining why a button is inactive. 

<button disabled dtt="This button is disabled">
  Disabled with tooltip
</button>

Anchor link tooltip

Use the att attribute on an anchor to show its href as a tooltip on hover.

Anchor tooltip 

<a att href="https://example.com">Anchor tooltip</a>

Utility

functora-css provides several utility attributes for quick styling without classes. These can be applied to any HTML element.

[full]

Sets width: 100% on the element.

<button full>Full width button</button>

[hidden]

Hides the element from the page. The element below has [hidden] and is not visible:

You cannot see this text.

<p hidden>You cannot see this text.</p>

Font families

Override the default font family with [serif], [sans], or [mono].

Serif font - [serif]

Sans-serif font - [sans]

Monospace font - [mono]

<p serif>Serif font</p>
<p sans>Sans-serif font</p>
<p mono>Monospace font</p>

Font sizes

Adjust font size with the [fs] attribute. Available values: xs, s, m, l, xl.

Extra large - [fs="xl"]

Large - [fs="l"]

Medium - [fs="m"]

Small - [fs="s"]

Extra small - [fs="xs"]

<p fs="xl">Extra large</p>
<p fs="l">Large</p>
<p fs="m">Medium</p>
<p fs="s">Small</p>
<p fs="xs">Extra small</p>
