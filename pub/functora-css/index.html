functora-css

A classless, JS-free, dependency-free, configuration-free, mobile-first, responsive, lightweight, semantic, pure CSS framework with light and dark themes and a collapsible navbar.

This page is built entirely with semantic HTML elements and styled exclusively by functora-css - no extra styles, no JavaScript.

Appearance

functora-css supports light and dark themes with pure CSS - no JavaScript required. Three independent mechanisms work together: system preference, an HTML attribute, and a hidden checkbox. Each is suited to a different use case.

1. Automatic - follow system preference

No special markup is needed. The framework reads prefers-color-scheme automatically. Best for blogs, documentation, and any page where matching the OS theme is sufficient. 

<!doctype html>
<html lang="en">
  <head>
    <link href="functora.min.css" rel="stylesheet" />
  </head>
  <body>
    <h1>Automatic theme</h1>
  </body>
</html>

2. Stateful - attribute (websites, apps)

For websites and applications that store the user's preference in a database, session, or localStorage. Set functora-theme-dark on <html> server-side or via JavaScript. The attribute overrides the system preference: 

<html functora-theme-dark>
<!-- your app manages this attribute from stored preference -->

3. Stateless - checkbox (static JS-free pages)

Place a hidden checkbox and a visible label anywhere on the page. The toggle is pure CSS and persists only for the current page view. No JavaScript, no storage needed. The label shows 🌚 or 🌝 automatically.

Inside the navbar - as shown on this page. The checkbox is the last child of <nav>. The label sits inside <ul> as a nav item: 

<nav>
  <label>
    <header>
      <a href="#">Site</a>
      <a href="#functora-nav-menu" id="functora-nav-open"></a>
    </header>
    <a href="#functora-nav-close" id="functora-nav-close"></a>
    <ul>
      <li><a href="/">Home</a></li>
      <li><label for="functora-theme-dark">Theme</label></li>
    </ul>
  </label>
  <input type="checkbox" id="functora-theme-dark" />
</nav>

Standalone (no navbar) - place the pair anywhere in the document:

Add checked for dark by default:

<input type="checkbox" id="functora-theme-dark" checked />
<label btn for="functora-theme-dark">Theme</label>

Omit checked for light by default:

<input type="checkbox" id="functora-theme-dark" />
<label btn for="functora-theme-dark">Theme</label>

Navigation

Pure CSS responsive navbar - no JavaScript. Two expand-collapse mechanisms are available. Choose based on your use case.

Anchor-based (static pages)

Uses the :target pseudo-class. Add id="functora-nav-menu" to the nav. <a href="#functora-nav-menu"> opens the menu, <a href="#functora-nav-close"> closes it. Clicking any anchor link inside the menu automatically closes the navbar because the URL hash changes away from #functora-nav-menu. This auto-close behavior is unique to the anchor-based approach - no JavaScript required. Best for static JS-free sites where navigation uses hash links. 

<nav id="functora-nav-menu" aria-label="Navbar">
  <label>
    <header>
      <a href="#">Site</a>
      <a href="#functora-nav-menu" id="functora-nav-open"></a>
    </header>
    <a href="#functora-nav-close" id="functora-nav-close"></a>
    <ul>
      <li><a href="/">Home</a></li>
      <li><a>Dropdown</a>
        <menu>
          <menuitem><a href="/a">Item</a></menuitem>
        </menu>
      </li>
      <li><label for="functora-theme-dark">Theme</label></li>
    </ul>
  </label>
  <input type="checkbox" id="functora-theme-dark" />
</nav>

Checkbox-based (SPAs and stateful sites)

Uses the :checked pseudo-class. Add a hidden <input type="checkbox" id="functora-nav-toggle"> inside the <label> wrapper. The close overlay is a <span> (not an <a>), avoiding hash conflicts with client-side routers. Note that clicking a nav link does not close the menu automatically - the user must tap the close overlay. If automatic collapse is desired, uncheck #functora-nav-toggle via JavaScript. Best for SPAs and stateful websites where JavaScript is already present and hash-based routing would clash with anchor nav. 

<nav aria-label="Navbar">
  <label>
    <input type="checkbox" id="functora-nav-toggle" />
    <header>
      <a href="#">App</a>
      <span id="functora-nav-open"></span>
    </header>
    <span id="functora-nav-close"></span>
    <ul>
      <li><a href="/">Home</a></li>
      <li><label for="functora-theme-dark">Theme</label></li>
    </ul>
  </label>
  <input type="checkbox" id="functora-theme-dark" />
</nav>

Fixed navbar

Add the fx attribute to fix the navbar to the top of the viewport. 

<nav fx>
  <!-- content -->
</nav>

Forms

Form elements are styled with a clean, minimal look. Labels should be placed directly before their corresponding inputs. Text inputs and textareas default to 100% width.

Text inputs

Text inputs have a subtle border that highlights on focus. Place the <label> directly before the input. 

<label for="name">Name</label>
<input id="name" type="text" placeholder="Your name" />

Textarea

Textareas auto-resize vertically and have the same styling as text inputs. 

<label for="bio">Bio</label>
<textarea id="bio" placeholder="Tell us about yourself"></textarea>

Select

Select dropdowns are styled to match other form elements.

  <option>Rust</option>

  <option>Python</option>

Radio and checkbox

<input type="radio" name="radio" id="r2" checked /><label for="r2">Option B</label>

Utility

functora-css provides several utility attributes for quick styling without classes. These can be applied to any HTML element.

xx - bypass class exclusion

By default, functora-css only styles elements without a class attribute (using :not([class])). Add xx to an element with a class to restore functora-css styling. This is useful when integrating with other frameworks or component libraries. 

<div class="my-component" xx>
  <!-- styled by both functora-css and .my-component -->
</div>

full - full width

Sets width: 100% on the element.

<button full>Full width button</button>

m-full - full width on mobile

Sets width: 100% at mobile viewport sizes only. The element retains its natural size on desktop. 

<button m-full>Full on mobile</button>

hidden - hide element

Hides the element from the page. The element below has hidden and is not visible:

You cannot see this text.

<p hidden>You cannot see this text.</p>
