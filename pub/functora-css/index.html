functora-css

A classless, JS-free, dependency-free, configuration-free, mobile-first, responsive, lightweight, semantic, pure CSS framework with light and dark themes and a collapsible navbar.

This page is built entirely with semantic HTML elements and styled exclusively by functora-css - no extra styles, no JavaScript.

Appearance

Three mechanisms for light/dark themes: system preference, HTML attribute, or checkbox.

1. Automatic

No markup needed. Automatically matches the OS theme.

<!doctype html>
<html lang="en">
  <head>
    <link href="functora.min.css" rel="stylesheet" />
  </head>
  <body>
    <h1>Automatic theme</h1>
  </body>
</html>

2. Attribute

Set functora-theme-dark on <html> to override system preference. Use when preference is stored server-side or in localStorage. 

<html functora-theme-dark>
  <!-- content -->
</html>

3. Checkbox

Place a hidden checkbox and a visible label anywhere on the page. No JavaScript, no storage needed. The label shows 🌝 or 🌚 automatically.

Inside the navbar — checkbox as last child of <nav>, label inside <ul> as a nav item: 

<nav>
  <label>
    <header>
      <a href="#">Site</a>
    </header>
    <ul>
      <li><a href="/">Home</a></li>
      <li><label for="functora-theme-dark">Theme</label></li>
    </ul>
  </label>
  <input type="checkbox" id="functora-theme-dark" />
</nav>

Standalone (no navbar) - place the pair anywhere in the document:

Add checked for dark by default:

<input type="checkbox" id="functora-theme-dark" checked />
<label btn for="functora-theme-dark">Theme</label>

Omit checked for light by default:

<input type="checkbox" id="functora-theme-dark" />
<label btn for="functora-theme-dark">Theme</label>

Navigation

Pure CSS responsive navbar - no JavaScript. Two expand-collapse mechanisms are available. Choose based on your use case.

1. Anchor

Add id="functora-nav-expand" to the nav. <a href="#functora-nav-expand"> opens the menu, <a href="#functora-nav-collapse"> closes it. The menu auto-closes when clicking any nav link. No JavaScript required. Best for static JS-free sites. 

<nav id="functora-nav-expand" aria-label="Navbar">
  <label>
    <header>
      <a href="#">Site</a>
      <a href="#functora-nav-expand" id="functora-nav-open"></a>
    </header>
    <a href="#functora-nav-collapse" id="functora-nav-collapse"></a>
    <ul>
      <li><a href="/">Home</a></li>
      <li><a>Dropdown</a>
        <menu>
          <menuitem><a href="/a">Item</a></menuitem>
        </menu>
      </li>
      <li><label for="functora-theme-dark">Theme</label></li>
    </ul>
  </label>
  <input type="checkbox" id="functora-theme-dark" />
</nav>

2. Checkbox

Add <input type="checkbox" id="functora-nav-toggle"> inside <label>. Use <span> for the close overlay to avoid hash conflicts with client-side routers. The menu does not auto-close on link click. Best for SPAs and stateful sites using JavaScript. 

<nav aria-label="Navbar">
  <label>
    <input type="checkbox" id="functora-nav-toggle" />
    <header>
      <a href="#">App</a>
      <span id="functora-nav-open"></span>
    </header>
    <span id="functora-nav-collapse"></span>
    <ul>
      <li><a href="/">Home</a></li>
      <li><label for="functora-theme-dark">Theme</label></li>
    </ul>
  </label>
  <input type="checkbox" id="functora-theme-dark" />
</nav>

Fixed navbar

Add the fx attribute to fix the navbar to the top of the viewport. 

<nav fx>
  <!-- content -->
</nav>

Dropdown menus

Nest a <menu> inside a <li>. The menu appears on hover (desktop) or as part of the mobile drawer.

  <menu>

    <menuitem><a href="/b">Item B</a></menuitem>
  </menu>
</li>
